「店長の聖域」接続・データ仕様書

1. アクセス制御 (Gatekeeper)
パス: src/app/dashboard/sanctuary/page.tsx
有効化フラグ: tenant_flags.mental_log_enabled が true の場合のみ表示。false の場合は 404 または設定ページへの誘導を行う。
認証: verifyAdmin() 必須。

2. データ抽象化レイヤー (AI Prescription API)
生ログを直接取得するのではなく、サーバーサイドで以下の処理を行った結果のみをフロントエンドに渡します。
処理場所: src/app/dashboard/sanctuary/actions.ts (Server Action)
データ取得フロー:
Main DB の logs テーブルから暗号化されたログを抽出（直近7日間など）。
サーバーサイドでのみ、ENCRYPTION_KEY を用いてログを復号。
復号された生テキストを AI（LLM）に投入。
AIが以下の項目で要約・分析（処方箋）を生成。
組織のバイタル:（例：活気 70%, 疲労 30%）
主なトピック:（例：シフトの柔軟性、備品の不足）
店長への提言:（例：休憩スペースの改善を検討してください）
生ログを破棄し、生成された「処方箋」オブジェクトのみをクライアントに返却。

3. フロントエンド表示項目 (Mild Presentation)
店長の画面には、以下の抽象化された情報のみが表示されます。
トレンドグラフ: 心理的安全性の推移（数値化されたもの）。
ワードクラウド / 主要トピック: 具体的な発言者を除去した「キーワード」。
AI処方箋カード: AIによる改善アドバイス。
